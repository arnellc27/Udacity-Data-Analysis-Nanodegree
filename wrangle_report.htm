Wrangle Report

By Chris Arnell

April 2021

This report details the steps taken in completing the Data Wrangling project as part of the Udacity Data Analysis Nanodegree.

Gathering

Data required to complete this project was detailed in three different sources. An Archive of twitter data was given as a CSV file and this was simply read into my notebook. Data predicting the breed of each dog was available to download for the Udacity servers and this was gathered using a requests function. Finally, and the most challenging method of data gathering was to collect the tweet data direct from twitter using an API. This was the most time consuming as the Twitter API limited the amount of data that could be pulled in a certain amount of time.

Assess

Assessing the data was split into 2 sections a visual assessment and a programmatic assessment.

Visual Assessment to carry out a visual assessment I used Excel to explore the 3 datasets. By doing this I could see the size of the data, 2537 rows which was manageable in Excel. Also, this allowed me to get a feel for the data, what data is given whilst also spot obvious errors should as spelling mistakes or unexpected values.

Programmatic assessment A programmatic assessment of the data was carried out using pandas functions such as .head(), .describe(), .sample() & .value_counts(). At the same time, I checked for any duplicate or unsignificant data to allow me to simplify my data frames.

During the assessment phase, any findings were categorised as either Quality, or tidiness issues

A quality issue is defined as either a problem with completeness, validity, accuracy or consistency. Examples of quality issues I found were:

1.     Different number of records across the 3 datasets

2.     Incorrect datatype for given information

3.     Rating denominator column containing unexpected values

A Tidiness issue is defined as a structural problem. Examples of this I found were:

1.     Columns duplicated across the 3 datasets

2.     The dog stage variable was spread over 4 columns

 

Clean

The final step in data wrangling was to clean the data. To do this I used the standard process of Define; to identify what process needs to be carried out, Code; write the code to carry out the required actions, Test; to verify that the code has had the desired result. Most of the cleaning was carried out using programmatic functions such as merge & melt. In some instances, manual cleaning was also required to correct data previously extracted programmatically.

One these steps have been carried out we have a clean, easy to read data frame that can be used for all future analysis
